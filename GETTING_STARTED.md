# دليل البدء السريع - بوليفارد صنعاء

## الخطوات الأولى عند التشغيل

### 1. تثبيت المتطلبات
```bash
# تثبيت جميع المكتبات
npm install
# أو باستخدام pnpm
pnpm install
```

### 2. إعداد متغيرات البيئة
أنشئ ملف `.env.local` في جذر المشروع وأضف:
```env
# قاعدة البيانات (اختياري - سيستخدم SQLite افتراضياً)
DATABASE_URL=sqlite:./dev.db

# OAuth (مطلوب للإنتاج فقط)
OAUTH_SERVER_URL=https://api.manus.im
VITE_APP_ID=your_app_id
JWT_SECRET=your_jwt_secret

# معلومات المالك (اختياري)
OWNER_NAME=بوليفارد صنعاء
OWNER_OPEN_ID=owner_id
```

### 3. تشغيل المشروع
```bash
# تشغيل خادم التطوير
npm run dev
# أو
pnpm dev
```

سيبدأ الخادم على: `http://localhost:3000`

### 4. الوصول للتطبيق
- **الصفحة الرئيسية**: http://localhost:3000/
- **صفحة الحجوزات**: http://localhost:3000/bookings
- **لوحة التحكم الإدارية**: http://localhost:3000/admin
- **ماسح QR**: http://localhost:3000/qr-code-scanner

## اختبار إنشاء حجز جديد

### الطريقة الأولى: عبر الواجهة الرسومية
1. اذهب إلى `/bookings`
2. انقر على زر "إضافة حجز جديد"
3. ملء البيانات:
   - **الاسم**: أدخل اسم العميل
   - **الهاتف**: رقم الهاتف
   - **نوع الخدمة**: اختر (شاليه / قاعة / كليهما)
   - **تاريخ الحجز**: اختر التاريخ
   - **عدد الضيوف**: أدخل العدد
   - **السعر الإجمالي**: أدخل السعر
4. انقر "حفظ الحجز"

### التحقق من الحفظ في قاعدة البيانات
بعد إنشاء الحجز مباشرة:
1. ستظهر رسالة نجاح: "تم حفظ الحجز بنجاح"
2. سيظهر الحجز في قائمة الحجوزات
3. يمكنك طباعة الحجز كـ PDF
4. يمكنك عرض تفاصيل الحجز الكاملة

### التحقق المتقدم: فحص ملف قاعدة البيانات
```bash
# فتح قاعدة البيانات SQLite
sqlite3 dev.db

# عرض جميع الحجوزات
SELECT * FROM bookings;

# عرض حجز محدد
SELECT * FROM bookings WHERE id = 1;

# عرض عدد الحجوزات
SELECT COUNT(*) FROM bookings;

# الخروج
.exit
```

## استكشاف الأخطاء

### خطأ: "Database not available"
**الحل:**
```bash
# تأكد من وجود ملف dev.db
ls -la dev.db

# إذا لم يكن موجوداً، سيتم إنشاؤه تلقائياً عند التشغيل
npm run dev
```

### خطأ: "Cannot find module 'better-sqlite3'"
**الحل:**
```bash
# أعد تثبيت المكتبات
rm -rf node_modules package-lock.json
npm install
```

### خطأ: "Port 3000 is already in use"
**الحل:**
```bash
# غير المنفذ في ملف التكوين أو استخدم:
PORT=3001 npm run dev
```

### الحجز لا يظهر بعد الحفظ
**الحل:**
1. تحقق من أن قاعدة البيانات متصلة
2. افتح console المتصفح (F12) وتحقق من الأخطاء
3. تحقق من أن البيانات المدخلة صحيحة
4. أعد تحميل الصفحة (F5)

## الملفات المهمة

| الملف | الوصف |
|------|-------|
| `server/db.ts` | دوال قاعدة البيانات |
| `server/routers.ts` | API endpoints |

## نصائح مهمة

1. **حفظ البيانات**: جميع الحجوزات تُحفظ تلقائياً في `dev.db`
2. **النسخ الاحتياطية**: انسخ ملف `dev.db` قبل حذفه
3. **الأداء**: قاعدة البيانات SQLite مناسبة للتطوير فقط
4. **الإنتاج**: في الإنتاج، استخدم MySQL أو PostgreSQL
